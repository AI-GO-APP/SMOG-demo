## 快速開始

兩個 demo 都是純 HTML/CSS/JS，**雙擊 `index.html` 即可在瀏覽器開啟**，不需要安裝任何東西。

| Demo | 檔案 | 主要展示 |
|---|---|---|
| **A · 排班時間軸** | `demo-a/index.html` | 以車為單位的時間軸、新增案件、Top 3 建議引擎 |
| **B · 路線地圖** | `demo-b/index.html` | 各車當日路線、OSRM 路徑、站點地圖 |

> **Demo B 注意事項：** 路線計算需要連網（呼叫 OSRM 公開 API）。若無網路，會自動降級為直線估算（虛線顯示）。

## Demo A 功能說明

**核心展示：智能排班建議引擎**

操作流程：
1. 進畫面看到 5 台車今日排班時間軸（含 14 筆模擬案件）
2. 點任一案件方塊可看詳細資料、改狀態
3. 點「+ 新增案件」填表單
4. 系統根據距離、時間空檔、金額三個權重，計算 Top 3 適合插入位置
5. 可在側欄調整權重百分比，即時影響建議結果
6. 待排清單側欄也可一鍵試算建議

**建議引擎邏輯：**
- 過濾車輛類型：贊量車不接施作、施作車不接贊量
- 檢查時間衝突
- 估算行車時間（簡化為 30 km/h + 5 分 buffer）
- 計算分數 = w1·距離分 + w2·空檔分 + w3·金額分

**模擬資料規格：**
- 5 台車（兩用 ×2、施作 ×2、贊量 ×1）
- 14 個今日案件（三種類型混合）
- 3 個待排案件
- 10 個示範地址（桃園+雙北）

## Demo B 功能說明

**核心展示：路線視覺化**

操作流程：
1. 進畫面預設顯示車 1 路線
2. 左側欄切換車輛 → 地圖即時更新
3. 點地圖編號 pin 看案件詳情
4. 點下方站點清單會 zoom 到該點並開啟 popup
5. 上方面板顯示總距離、預估行車時間

**技術細節：**
- 地圖：Leaflet + OpenStreetMap（完全免費、無 API key）
- 路線：OSRM 公開 server `router.project-osrm.org`
- 路線失敗時降級為直線（灰色虛線）

## 設計決策說明（給工程師看）

**為什麼用模組化分檔（而不是 React/Vue）？**
- Demo 階段優先「快速改、給主管看」
- 沒有 build step、沒有 npm install
- 雙擊就能跑，回饋週期最短
- 方向確認後再轉到 Next.js + TypeScript 不會浪費，因為每個元件邊界已清楚

**為什麼用 IIFE pattern？**
- 避免污染全域 scope
- 每個模組只暴露公開 API
- 等同 ES module 的 export 行為，但相容 file:// 開啟

**為什麼 Demo A、Demo B 各有 mock-data.js？**
- 兩個 demo 獨立執行，方便分發
- 真實開發應抽出共用 module
